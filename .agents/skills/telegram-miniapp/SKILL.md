# Telegram Mini‑App Skill

## When to Use
- You want to deliver a full‑featured web application that runs inside Telegram as a **Mini‑App** (Web App). 
- You need a quick starter project with modern UI, SEO, and secure integration with the Telegram Bot API.
- You prefer a vanilla‑HTML/JS stack (no framework) unless explicitly requested.

## Overview
A Telegram Mini‑App is essentially a regular web page loaded inside Telegram via the `Web App` platform. The page can interact with Telegram through the `Telegram.WebApp` JavaScript object, allowing you to:
- read user data (`initData`),
- request contact information,
- open the main bot chat,
- send data back to the bot via `sendData`.

This skill sets up a **minimal but premium** project skeleton, includes:
1. Project structure (`index.html`, `style.css`, `app.js`).
2. A simple build script using **Vite** (optional, only if you ask for a framework later).
3. Deployment notes for hosting on a static site (e.g., GitHub Pages, Netlify).
4. Guidance on securing the `initData` signature verification on the server side (Python Flask example).
5. Rich UI guidelines (glassmorphism, dark mode, Google Fonts) to meet the aesthetic standards.

## Steps to Create a Mini‑App

### 1. Bot Setup
1. Create a bot with **@BotFather** and obtain the **Bot Token**.
2. In Bot Settings enable **“Web Apps”** and set the **Domain** where the Mini‑App will be hosted.
3. Save the **Bot Token** and **Domain** – you’ll need them for server‑side verification.

### 2. Project Layout
Create the following Vite‑React‑TypeScript scaffold under your desired root folder (e.g., `miniapp/`). Use the standard Vite command `npm create vite@latest miniapp -- --template react-ts` and then install dependencies.

After scaffolding, replace the default `src/App.tsx` and `src/index.css` with the premium UI code from the original skill (glassmorphism, dark mode, Google Font "Inter"). The `public/index.html` will be automatically generated by Vite; you only need to add the `<meta>` tags for SEO inside it.

### 5. `app.js`
```js
// Initialise Telegram Web App
const tg = window.Telegram.WebApp;

// Populate user name from init data if available
if (tg.initDataUnsafe?.user?.first_name) {
  document.getElementById('user-name').textContent = tg.initDataUnsafe.user.first_name;
```

### 6. Server‑Side `verify.py` (Python Flask example)
```python
import os, hmac, hashlib, base64
from flask import Flask, request, abort, jsonify

app = Flask(__name__)
BOT_TOKEN = os.getenv('BOT_TOKEN')  # set this env var on your server
SECRET = hashlib.sha256(BOT_TOKEN.encode()).digest()

def verify_init_data(init_data: str) -> bool:
    # split key=value pairs, the last pair is hash
    *data_pairs, hash_pair = init_data.split('\n')
    data_str = '\n'.join(data_pairs)
    check_hash = hash_pair.split('=')[1]
    calc_hash = hmac.new(SECRET, data_str.encode(), hashlib.sha256).hexdigest()
    return hmac.compare_digest(calc_hash, check_hash)

@app.post('/verify')
def verify():
    init_data = request.form.get('init_data')
    if not init_data or not verify_init_data(init_data):
        abort(400, 'Invalid init data')
    return jsonify(status='ok')

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8080)
```
- Shows how to validate the signature using the Bot Token.

### 7. Deployment Checklist
- Host the static files (`index.html`, `style.css`, `app.js`) on HTTPS (required by Telegram).
- Add the domain to **Bot Settings → Web Apps**.
- If you use the server verification, expose `/verify` endpoint with a valid TLS cert.
- Set `BOT_TOKEN` as an environment variable on the server.
- Test by opening the link inside a Telegram chat (`t.me/YourBot?start=webapp`).

## Best‑Practice Checklist
- **Security**: Never expose the Bot Token on the client side.
- **Performance**: Minify CSS/JS, enable HTTP/2, use CDN.
- **Accessibility**: Use semantic HTML, proper `aria` labels for buttons.
- **Responsive**: Ensure the layout works on mobile screens (max‑width 320px).
